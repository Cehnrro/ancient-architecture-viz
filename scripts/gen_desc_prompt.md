# 批量生成中国古代建筑简介

请根据提供的中国古代建筑名单，为每一处建筑撰写一条可直接用于展览展示、文博介绍或建筑说明场景的专业简介，并以 JSON 格式输出。

【最终输出格式】
{
  "建筑名称": "建造年代：……；所在地区：……；建筑特点：……；历史文化价值：……。",
  "建筑名称2": "建造年代：……；所在地区：……；建筑特点：……；历史文化价值：……。"
}

【硬性格式要求】
1. 输出内容必须为合法 JSON，对象的 key 为建筑名称，value 为对应简介。
2. 每条简介必须严格包含并按以下顺序呈现四个字段，不可缺失、不可调换顺序：
   - 建造年代：
   - 所在地区：
   - 建筑特点：
   - 历史文化价值：
3. 四个字段之间统一使用全角分号“；”连接，结尾使用句号“。”。
4. 不要输出标题、说明文字、编号、注释、前言、后记或 markdown 标记，只输出最终 JSON。
5. 不要把四个字段拆成数组、子对象或换行格式，必须写成单条完整字符串，便于直接展示和导入系统。

【内容要求】
1. 建造年代：
   - 应尽量写明始建年代、创建朝代、主要营建时期、重修时代或现存主体所属时代。
   - 若具体年份不详，可写“始建于某朝”或“现存主体为某代遗构”。
   - 若建筑历经多次修缮，应优先突出“始建时代”和“现存主要形制所反映的时代”。

2. 所在地区：
   - 应尽量具体到“省+市+县（区）”。
   - 若资料有限，至少写清省市或明确地理归属。
   - 避免只写模糊地名，如“江南地区”“中原一带”。

3. 建筑特点：
   - 这是重点部分，必须具体展开，不能泛泛而谈。
   - 应结合建筑类型，尽量描述以下一个或多个方面：
     - 总体布局
     - 平面组织
     - 中轴关系
     - 结构体系
     - 梁架做法
     - 屋顶形式
     - 材料使用
     - 装饰工艺
     - 雕刻彩绘
     - 空间功能
     - 地域风格
     - 防御、祭祀、居住、交通、教学、商贸等功能特征
   - 应尽量写出该建筑区别于同类建筑的辨识度，不要只写“造型优美”“工艺精湛”“风格独特”这类空泛表述，除非后面有具体说明。

4. 历史文化价值：
   - 这是重点部分，必须结合建筑本身的历史功能、文化背景和保存意义展开说明。
   - 可参考百度百科等公开资料中的史实信息，但必须重新整合、概括、转述，不可照抄。
   - 应尽量落到具体层面，例如：
     - 地方政务与治理
     - 宗教传播与信仰活动
     - 书院教育与学术传统
     - 商帮往来与会馆文化
     - 家族制度与礼制传统
     - 军事防御与边地治理
     - 交通发展与桥梁工程
     - 地域营造技艺传承
     - 民俗生活与地方社会结构
     - 园林审美与文人文化
     - 建筑史、工艺史、地方史研究价值
   - 如果查不到充分、可靠的信息来源，可谨慎少写，只写能够确认的保存意义、类型价值或研究意义，绝不能凭空编造。

【篇幅要求】
1. 不设置字数上限。
2. 内容应尽量详细、完整，适合展示使用。
3. 但不允许为了拉长篇幅而堆砌空话、套话或重复表述。
4. 每个字段都应写成完整通顺的说明句，不能写成过于简略的词语拼接。

【去重复要求】
1. 严禁批量生成时反复使用相同句式、相同结尾或相同价值判断。
2. 尤其不要机械重复以下空泛说法：
   - “具有重要的历史文化价值”
   - “是研究某地历史的重要实物资料”
   - “具有较高的艺术价值和研究价值”
   - “体现了古代劳动人民的聪明才智”
3. 如确需表达价值，必须结合建筑具体类型、地域背景、功能属性和历史作用展开，不得只给结论，不给内容。
4. 不同类型建筑必须写出差异：
   - 宫殿、坛庙、衙署：突出礼制、权力秩序、制度空间
   - 寺庙、道观、塔幢：突出宗教活动、信仰传播、造像或空间营造
   - 书院、学宫：突出教育功能、文脉传承、学术传统
   - 民居、庄园、故居、祠堂：突出家族制度、居住格局、地方风俗
   - 会馆、戏台、商铺：突出商贸交流、行业组织、地域移民文化
   - 桥梁、城墙、门楼、驿道：突出交通、防御、工程技术与区域联系
   - 园林、亭台楼阁：突出审美意趣、游赏功能与文人文化

【真实性要求】
1. 不使用“据说”“相传”“可能”“大概”“或为”等不确定且未经证实的表述。
2. 对无法确认的史实，不要强行补全。
3. 若某一项信息不足，可采取审慎表述，例如：
   - 建造年代写已知朝代或现存时代
   - 历史文化价值写可确认的保存价值、类型价值、研究价值
4. 宁可信息保守、准确，也不要虚构细节。

【语言要求】
1. 语言应正式、专业、准确，适合文博、展陈、地方文化宣传和古建筑介绍场景。
2. 避免口语化表达。
3. 避免华丽空洞修辞。
4. 以客观介绍为主，不写主观赞美。
5. 同一批内容中尽量避免句法结构高度雷同。

【示例风格】
"某建筑名称": "建造年代：始建于明代，现存主体建筑多为清代修缮遗存。所在地区：位于江西省吉安市某县。建筑特点：整体布局规整，沿中轴组织主要空间，采用传统木构架体系，梁架做法完整，屋面形式和细部构件保存较好，雀替、枋梁及木雕装饰具有鲜明的地方工艺特征，体现出当地传统公共建筑的营造方式。历史文化价值：该建筑不仅反映了特定时期地方营建技术和审美取向，也承载了区域社会组织、礼仪活动与公共生活的历史信息，对研究当地建筑演变、社会结构及地域文化传统具有较为具体的实物意义。"

【执行原则】
- 优先保证准确性，其次保证完整性，再考虑文采。
- 优先写出建筑自身最有辨识度的内容，不要套用统一模板。
- 文化价值部分必须因建筑而异，不可千篇一律。
- 若资料不足，少写、稳写，不要乱写。
